# Core Principles (憲法)

## 1. Simplicity & Minimalism (単純さとミニマリズム)
- **Keep it Simple**: 複雑なアーキテクチャよりも、理解しやすくメンテナンスしやすい単純な構造を優先する。
- **YAGNI (You Aren't Gonna Need It)**: 今必要な機能だけを実装する。将来必要になるかもしれない機能は、必要になった時に追加する。
- **Less is More**: コード量、ファイル数、依存ライブラリを最小限に抑える。

## 2. Clarity & Transparency (明瞭さと透明性)
- **Explicit over Implicit**: 暗黙的な挙動よりも、明示的な記述を好む。
- **Document Decisions**: なぜそのアプローチを選んだのか、その決定背景を記録に残す。
- **Single Source of Truth**: 情報の重複を避け、常に一つの情報源を参照するようにする。

## 3. Iterative Improvement (反復的な改善)
- **Start Small**: 小さく始め、フィードバックを得て改善するサイクルを回す。
- **Fail Fast**: 問題があれば早期に発見し、修正する。

## 4. Human-Centric (人間中心)
- **Readable Code**: コードは機械のためだけでなく、人間（将来の自分やチームメンバー）のために書く。
- **Meaningful Naming**: 変数名、関数名、ファイル名は、その役割を明確に表すものにする。
