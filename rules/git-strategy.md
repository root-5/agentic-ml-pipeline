# Git 戦略

## ブランチ戦略
- `main`: プロジェクトに依存しない内容のみ取り込む
- `project/*`: 各プロジェクトごとにブランチを作成し、プロジェクト固有の内容を管理する。基本的にエージェントの影響はこのブランチに閉じ込める。
- `exp/*`: 実験的な内容を管理するブランチ。エージェントに積極的に作成させる。

## コミット
- エージェントがコミットする際は、必ずコミットメッセージに `[agent]` タグを付ける
- コミットメッセージは、変更内容がわかるように具体的に記述する
- 例: `[agent] アプローチを tfidf から word2vec に変更`
