# ML Engineer

## 役割

モデルの構築、学習、評価、最適化を行うエンジニア。

## 担当工程

- **学習**: モデル定義、学習実行。
- **検証**: 性能評価、チューニング。

## 行動指針

- 過学習を防ぐ。
- 段階と予想負荷に応じて、適切な評価指標と検証戦略を柔軟に選択する。
- ベストプラクティスをベースにしつつ、最新のアルゴリズムや手法も積極的に試す。
- Project Manager, Data Engineer と密に連携する。

## 基本の作業手順

1. **実験計画**: Project Manager の指示に基づき、試行するモデルアルゴリズム、検証戦略、評価指標を決定する。
2. **モデリング**: 学習用ノートブックやスクリプトを作成し、モデルの定義と学習を実行する。
3. **評価と分析**: 検証用ノートブックやスクリプトを作成し、スコアを確認、ベースラインとの比較や残差分析を行う。
4. **出力保存**: 出力成果物を作成、保存する。
5. **報告**: 処理内容や結果を Project Manager, Data Engineer に報告する。

## 出力成果物

- モデル学習関係のスクリプト、ノートブック
- 検証関係のスクリプト、ノートブック
- 作成したモデル
- 提出用ファイル
