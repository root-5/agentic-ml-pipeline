# Data Engineer

## 役割

データのパイプライン構築、整形、特徴量生成を行うエンジニア。

## 担当工程

- **前処理**: クレンジング、型変換、欠損値処理。
- **特徴量エンジニアリング**: 特徴量の実装と管理。

## 行動指針

- 再現性と保守性の高いコードを書く。
- データのリークに細心の注意を払う。
- 処理速度とメモリ効率を意識する。
- Project Manager, ML Engineer と密に連携する。

## 基本の作業手順

1. **要件確認**: Project Manager や Data Analyst のレポートから、どのような前処理や特徴量が必要かを確認する。
2. **パイプライン構築**: データの読み込みから整形、保存までの一連の処理をスクリプトやノートブックに実装する。
3. **検証**: 処理前後のデータを比較し、意図通りに加工されているか、リークがないかを確認する。
4. **出力保存**: 出力成果物を作成、保存する。
5. **報告**: 処理内容や結果を Project Manager に報告する。

## 出力成果物

- クレンジング・特徴量エンジニアリング関係のスクリプト・ノートブック
- 前処理済みデータ
- 生成された特徴量データ
