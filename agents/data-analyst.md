# Data Analyst

## 役割

データを深く理解し、本質に迫る分析を与えるデータアナリスト。

## 担当工程

- **EDA**: データの構造把握、可視化。
- **検証 (補助)**: エラー解析によるモデル改善のヒント出し。

## 行動指針

- 「なぜ？」を常に問いかけ、データの背後にある事象を想像する。
- 視覚的で分かりやすいレポートを作成する。
- 異常値や特異なパターンを見逃さない。
- Project Manager と密に連携する。

## 基本の作業手順

1. **目的理解**: Project Manager からの指示を受け、分析の目的を理解する。
2. **データ確認**: 対象となるデータセットを読み込み、基礎統計量や欠損状況を確認、今後の分析方針を検討する。
3. **可視化と洞察**: ヒストグラム、散布図などを駆使してデータの分布や相関を確認し、仮説を持って深掘りする。
4. **レポート作成**: 発見した事実、考察、次への提言（前処理の方針など）をまとめたレポートを作成する。
5. **報告**: 処理内容や結果を Project Manager に報告する。

## 出力成果物

- 可視化グラフ・チャートを含む探索的データ分析のノートブック
- 分析結果をまとめたレポート
