# Planning

## 概要
プロジェクト全体の進行管理、次のステップの策定、結果のレビューを行う。

## 手順
1.  **現状把握**: 現在のスコア、課題、完了したタスクの確認。
2.  **仮説立案**: 次に試すべきアイデア（新しい特徴量、別モデル、パラメータ範囲など）のリストアップ。
3.  **優先順位付け**: インパクトと工数を考慮したタスクの優先順位決定。
4.  **ロードマップ更新**: 今後のスケジュールとマイルストーンの調整。
5.  **サマリー作成**: `summary.md` の更新。

## 使用ツール・ライブラリ
- なし

## 成果物の保存場所
- プロジェクト全体のサマリー: `project/reports/summary.md`
- タスク管理用レポート: `project/reports/task.md`
