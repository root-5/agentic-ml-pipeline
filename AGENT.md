# Agent Instructions

このファイルは、AI エージェントがこのプロジェクトに参加する際に最初に参照すべき指針書です。
あなたは `agents/` ディレクトリに定義された役割を担い、`skills/` にある手順を用いて、`rules/` に従いながらタスクを遂行します。

## ディレクトリ構造と読み方

このプロジェクトは以下の構成で成り立っています。エージェントは各ディレクトリのドキュメントを「知識」としてロードし、振る舞いを決定してください。

- **agents/**: エージェント定義。従業員のマネジメントのイメージで「役割・作業原則・作業パターン・作業完了条件」を定義。
    - ユーザーから役割（例: "Data Analystとして振る舞って"）を指定された場合、該当する `.md` ファイルを読み込み、その「役割」「行動指針」を模倣してください。
- **rules/**: ルール定義。社則のイメージで「全てのエージェント・どんなスキル領域でも従うべき規則」を定義。
    - どの役割、どのスキルを実行する場合でも、`rules/core-principles.md` および `rules/coding-standards.md` は常に順守してください。
- **{vendor}/skills/**: スキル定義。社内ドキュメントのイメージで「ドメイン知識・手順書・ツール使用法」を定義。
    - 特定のタスク（例: "EDAを実行して"）を依頼された場合、対応するディレクトリ内の `SKILL.md` を参照し、その手順に従ってください。
    - vendor は各エージェント提供者によります。 GitHub であれば `.github/`、claude であれば `.claude/` など。

## エージェント系マークダウン修正ルール

また、ここが汚れると全てのエージェントの動作に影響するため、これらのディレクトリ配下では、以下のルールを徹底する。

- シンプルかつミニマル
- 読み手に依存しない明瞭な記述
- 「とりあえず追加しとく」といった判断は絶対に避け、とにかく最低限から始める

## エージェントの基本動作フロー

1.  **役割の確立**: ユーザーの指示から適切な `agents/*.md` を特定し、ペルソナを設定する。
2.  **ルールの確認**: `rules/*.md` を読み込み、遵守事項を確認する。
3.  **スキルの選択**: タスクに必要な `skills/*/SKILL.md` を特定し、手順を把握する。
4.  **実行**: 手順に従いタスクを実行する。迷った場合は `rules/core-principles.md` の「シンプルかつミニマム」等の原則に立ち返る。
